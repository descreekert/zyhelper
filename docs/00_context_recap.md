## 现有 Excel FILTER 公式的核心逻辑

用户已在 Excel 中实现的筛选 + 排序逻辑（web 版要复现）：

### 1. 行类型识别（基于 `25参考可信度`）

```
25参考可信度="" 的行  →  停招行 (25 招过, 26 没招)
25参考可信度<>"" 的行 →  非停招行 (含 完全一致 / 有变化 / 新增)
```

### 2. 分通道筛选

| 筛选条件 | 非停招行用 | 停招行用 |
|---|---|---|
| 分数范围 | `25参考最低分` | `25最低分` |
| 学科评估 | `学科评估` (含括号格式) | `25学科评估` (纯评级) |
| 学费上限 | `学费` (26) | `25学费` |
| 选科要求 | `选科要求` (26) | `25选科要求` |

### 3. 通用筛选（所有行）

- `学校优先 ≤ 18`
- `专业类优先 ≤ 16`（仅非停招）

### 4. 选科要求语义

物理类志愿，化学必选，无生物/地理：
```
(选科要求 包含 "不限") OR
((选科要求 包含 "化学" OR "政治") AND 不包含 "生物" AND 不包含 "地理")
```

### 5. 停招开关

E8 = "含停招" / "不含停招"：控制是否在结果里包括停招行。

### 6. 排序

支持 7 种排序模式（默认按 25参考分数 降序，停招行用 25最低分 替代以正确穿插）：

- 城市-专业-学校 / 城市-学校-专业
- 专业-学校-城市 / 专业-城市-学校
- 学校-城市-专业 / 学校-专业-城市
- 默认（按分数降序）

## 用户痛点

1. **2026数据 sheet 81 列太宽**，横向滚动费眼
2. 想看某个分数段/位次段全集，要反复改 B6/G6 输入
3. 想多所学校对比，Excel 不易做
4. 学科评估列 `临床医学(A+),通信工程(B)` 含多个评级，用 D8 筛选效果不直观
5. 学制/学费/选科要求等细节，列宽不够，要展开/复制看

## 这次想做的 web app 核心

把上面的 FILTER 公式 + 排序 + 大量列展示，做成**响应式的卡片/列表/筛选侧栏**界面：
- 筛选条件做成侧栏（更直观）
- 结果以卡片或简表呈现，重要字段突出
- 详情/对比 用弹窗或并排卡片
- 一些数据可视化（一分一段定位、历年趋势）

详细见 `requirements.md` 和 `architecture.md`。
