## 2. 筛选逻辑 (filters.js)

### 2.1 单元测试

`scripts/test_filters.html` 写一个简单测试页（或用 Node 跑 ESM）：

```javascript
// 测试用例
import { applyFilters } from '../assets/filters.js';

const samplePlans = [
  /* 5 条手工构造的典型数据 (高分校 + 中外合作 + 停招 + 新增 + 无参考) */
];

const f = { scoreRange: [640, 660], ... };
const result = applyFilters(samplePlans, f);
assert(result.length === 2);
assert(result.every(p => p.ref25Score >= 640 && p.ref25Score <= 660));
```

### 2.2 关键 case 矩阵

| 场景 | 输入 | 期望 |
|---|---|---|
| 分数范围卡边界 | `[640, 650]` 包含 640 / 650 | 含端点 |
| 停招行 not included | `includeStopped=false` | 过滤掉所有 `isStopped` |
| 停招行 included | `includeStopped=true` | 全部出现 |
| 选科要求 | "生物" 必选的专业 | 必过滤 |
| 选科要求 | "化学和政治" 二选一 | 通过 |
| 选科要求 | "化学和生物" | 过滤（含生物）|
| 学科评估 全选 | 默认 | 不影响结果 |
| 学科评估 仅 A+ | 多评级行（A+,B,A）| 通过（含 A+ 即可） |
| 中外合作 关 | `includeMidOutside=false` | `isMidOutside` 全过滤 |
| 关键词 "电子信息" | | 含此关键词的学校/专业出现 |

### 2.3 性能基准

- 12k 行 × applyFilters → 应 < 30 ms（M1 mac）
- 全量排序 → 应 < 20 ms
- 拖动滑块到结果更新 → < 200 ms（含 debounce）
