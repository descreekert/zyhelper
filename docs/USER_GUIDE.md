## 0. 快速上手 (5 分钟)

1. 浏览器打开 https://descreekert.github.io/zyhelper/ (建议 Chrome / Edge / Safari)。
2. 顶部输入框填 **26 高考分数**(例: `620`)。**26 位次** 会从一分一段表自动反查。
3. 上方会出现 **志愿分布** 三档计数: `冲 N1: 分段(位次段) / 稳 N2 / 保 N3`。可点击该档进入筛选。
4. 左侧 **筛选面板** 选 城市 / 专业类 / 学校优先档 等。结果列表右侧出现匹配的招生计划。
5. 点列表中"➕ 志愿"把感兴趣的 plan 加进 **志愿单** (右上角 `📋 志愿(N)`)。
6. 切到 `📊 分析` 看冲稳保占比 / 录取趋势 / 转专业风险 等; 切到 `📄 PDF 报告` 一键生成完整 PDF 报告。

## 1. 主视图 (查询)

### 1.1 顶部工具栏

| 控件 | 说明 |
|----|----|
| **26 分数 / 26 位次** | 任填一个,另一个会按一分一段自动反查 |
| **基准** | 用哪一年的等位分换算 25 等位分: 25 (默认) / 平均 / 24 / 23 |
| **多年等位** | 显示当前 26 分数对应的 25/24/23 三年等位分位次,横向对比 |
| **志愿分布:冲/稳/保 chips** | 三档分数+位次范围。点击 chip 等于"仅看该档" |
| **⚙ 比例调节** | 调整冲/稳/保 目标比例 + 总志愿数 (默认 25%/45%/30%, 112 项) |
| **📋 / 🗂 / ⫷ / 🎯** | 视图模式: 表格 / 卡片 / 三栏分桶 / 推荐 |
| **📋 志愿(N)** | 切到"我的志愿单" |
| **🔗 / 📷 / 🖨 / 📥** | 分享/截图/打印/导入预设 |
| **🌙 / ⚙ 排序 / ⋯** | 主题 / 排序设置 / 更多菜单 |

### 1.2 筛选面板 (左侧)

- **关键字筛选** — 在结果上叠加 关键字 (按视图独立保存)
- **25 参考分数 / 位次 (冲稳保 3 段)** — 自动从 26 分数填充,可手调
- **学费上限**、**保研资格**、**985/211/双一流/省重点** 标签筛选
- **🏫 学校优先 / 🏙 城市优先 / 📚 专业类优先**: 按各档可选范围筛选,可在 `⚙ 排序` 中自定义档位边界
- **筛选预设**: 上方 `+ 保存当前` 把当前所有筛选另存为命名预设

### 1.3 列设置 (⋯ → 列管理)

可隐藏 / 显示 / 拖拽 列顺序,设置会持久化到 localStorage。每列可拖拽边线调整宽度。

### 1.4 排序

- 点击列表头 — 升降序切换
- `⚙ 排序` 弹出多键排序面板,可叠加多列 + 拖拽 reorder + 单独切方向

## 2. 志愿单 (右上角 `📋 志愿(N)`)

### 2.1 切换志愿单 + 多列表

- 顶部下拉框 — 切换不同志愿单 (例: "默认" / "650 分志愿" / "退而求其次")
- `新建` / `重命名` / `复制` / `删除` 操作旁边按钮
- 每个志愿单独立锁定/待确认/备份状态

### 2.2 排序与锁定

每行操作列:
| 按钮 | 含义 |
|----|----|
| `↑↓⤒⤓` | 向上 / 向下 / 置顶 / 置底 |
| `📌` | 锁定到当前位置 (整理时不被自动重排打乱) |
| `📌+` (锁所有) | 一键锁全部当前在档项 |
| `⏳` 待确认 (橙色) | 整理排序中的临时状态。`✓ 全部确认` 把所有 ⏳ 升为 📌; `↩ 全部撤销` 回滚到编辑前 |
| `🗑` | 移出志愿单 |

> 多选: 拖动选区选多行 / Ctrl 单击切换 / Shift 范围。选中后可批量上下移动。

### 2.3 自动排序规则

未锁定的 plan 按 **26 等位次升序** 排列 (高位次在前)。锁定的项停在原位置, 未锁项填空位。

### 2.4 导入 / 导出 dropdown

📤 **导出 ▾**
| 选项 | 用途 |
|----|----|
| HTML — 官网格式 | 模板化生成辽宁招生考试网正式版,可上传/打印 |
| HTML — 官网预选模式 | 模板化生成预选版,单表 |
| HTML — 简洁版 (打印) | 自定义简洁打印样式 |
| JSON | 全量导出 (含 plan ID,跨设备/版本可导入) |
| CSV | 6 列 (序号/院校代号/院校名称/专业代号/专业名称/专业备注) + UTF-8 BOM |
| CSV — 按 冲稳保 档 | CSV + 档 列 |

📥 **导入 ▾**
| 选项 | 用途 |
|----|----|
| HTML — 官网/预选 (自动识别) | 解析辽宁招考网导出的 HTML, 自动识别 5 批次 / 单表 |
| JSON | 还原 全量 |
| CSV | 行 = 1 志愿,自动按 院校代号+专业代号 / 院校名+专业名 匹配 |

**导入行为**:
- **空表 / 替换 / 新建** → 严格保留源文件顺序, 全部 📌 锁定
- **合并到非空表** → 新增项不锁定, 与现有项按 26 等位次自动重排

## 3. 分析页 (`📊 分析`)

### 3.1 总览 + Anchor 调整

- **26 分数 / 25 等位 / 26 实际预测位次** — 默认自动,可手调
- **🔥 忽略专业热度** checkbox — 计算录取率时把电子信息/计算机 等热度调整 (-4%/-3%) 当作 0
- 冲/稳/保/外 四张卡 — 点开看该档详细 plan 表 (含 ✓OK / ⚠ 需转 / ✗ 错 标记)

### 3.2 🏆 各档录取率 Top 3 + 🎯 阈值 Top 5

- **Top 3**: 冲/稳/保 三档 各自概率最高的 3 项 (含 学校 / 专业 / 概率 / Δ分位差)
- **Top 5**: 11 个阈值 (30%-80% 步 5%) × 5 列,展示该阈值下"按平行志愿顺序首 5 个" prob ≥ 阈值 的志愿

### 3.3 🎓 转专业风险扫描

- 2 个清单: **目标专业** (无需转) / **可接受转入**
- 默认内置 26 个目标 + 4 个可接受 (✎ 编辑清单 可改, 失焦自动保存; ⟲ 默认 重置)
- 3 张汇总卡: ✓ 无需转 / ⚠ 需转 / ✗ 不在范围 (点击展开看清单)
- 匹配规则:
  - **单专业**: 主名直接在 目标 / 可接受 → ok / warn
  - **大类** (如 `自动化类(自动化、测控、电子科学)`): 子专业含任一 目标 → ok; 否则含任一 可接受 → warn; 都没有 → error

### 3.4 📈 录取趋势图

- 橙色折线: 每个志愿的预测录取概率,X 轴 = 志愿顺序
- 蓝色虚线: 当前阈值 (默认 50%)
- 阈值首达点 (绿点): 每个阈值 (50%-95% 步 5%) 首个 prob ≥ 阈值 的志愿
- **互动**:
  - 拖动滑杆 (1% 步进, 0-99%)
  - 点 Y 轴数字 (0/20/40/60/80/100 快捷)
  - 点击概率分布柱状图 (10% 分桶)
  - 点 SVG 任意 Y 位置 → 精确设置
  - 鼠标移到曲线上 → tooltip 显示具体志愿 (含 Δ 分位次)
- 阈值下方动态表: 列出所有 prob ≥ 阈值 的志愿全表

### 3.5 录取率算法 (Sigmoid + 5 因子)

```
prob = clamp[ sigmoid(diff/SCALE + BIAS) + supplyAdj , 2%, 98% ]

diff   = effPlanRank - userRank25     (用户位次 vs plan 调整后位次)
effPlanRank = refRank × (1 + heatPct)
refRank = (ref25Rank + avgRank) / 2   (平均 50/50)
heatPct = 电子-4% / 计算-3% / 自动-1.5% / 机械-0.5% / 车-1% / 能动+1% / 其它 0
         (可在分析页 🔥 忽略专业热度 关闭)
SCALE = 1500 (位次差), BIAS = 0.4 (确保 diff=0 时 prob ≈ 60%)

supplyAdj:
  26 该分 总招生 / 26 该分 同分人数 > 0.85 → +10%
  > 0.5 → +5%
  else → 0
```

新增专业用 `ref25Score + 5` 反查位次代替 refRank, 标 `(估)`。

### 3.6 多维度聚合

- **按分数 (一分一段)** + **按学校** 并排
- **按报考专业** (主名收敛,例 `机械设计制造及其自动化(卓越)` 与 普通版合并)
- **按城市**
- **招生维度聚合**: 2026 全体 plans 在冲稳保整体范围内 各 25 分总招生

每个聚合可点行展开 → 看明细 plan 列表。

## 6. 常见问题

**Q: 输入 26 分数后位次没填?**
A: 检查 `data/score_rank.json` 是否含 2026 年一分一段 (页面顶部"数据生成"时间戳应是 2026 年)。

**Q: 录取率显示"新?"?**
A: 该 plan 是 2026 新增专业,无 25 年参考数据。算法用"该专业 25 ref 分 + 5"反查位次粗估,标 `(估)`。

**Q: 志愿表的"专业代码"在哪?**
A: 列管理 (⋯ → 列设置) 默认显示。代码 080202 等是教育部专业目录代码,跟 26 招生计划里的"专业代号" (例 Z1, B1) 不同。

**Q: 转专业判定为 ✗ 错,但该专业明明在我的目标里?**
A: 检查名称是否完全一致 (有时 "微电子科学与工程" 写成 "微电子科学与技术")。✎ 编辑清单 改名即可,LS 会自动迁移。

**Q: PDF 打出来跨页内容被截断?**
A: 浏览器选纸张 A4 + 边距 ≥ 8mm,关闭页眉页脚。Chrome 推荐。

**Q: 导入辽宁 HTML 志愿表为啥少了几条?**
A: 检查"未匹配 N 项"明细 — 通常是学校改名/专业名变更。可手动补加。

**Q: 我换电脑了,志愿单丢了?**
A: 导出 JSON → 新电脑 `📥 导入 ▾ JSON`,完整恢复 (含锁定/待确认状态)。

## 7. 隐私

- 所有数据 (志愿单 / 配置 / 锁定状态) 仅存浏览器 localStorage,不上传任何服务器
- 应用是纯静态 SPA, 部署在 GitHub Pages
- 你的高考分数/位次/志愿 完全本地,清空浏览器数据即清空
