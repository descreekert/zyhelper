# zyhelper 演示动图 — 3 个场景脚本

> 录制工具推荐: **LICEcap** (https://www.cockos.com/licecap/) 或 **Kap** (https://getkap.co/)
> 也可用 macOS QuickTime Player → 录制 .mov, 再用 ffmpeg 转 gif
> 浏览器: Chrome / Edge,窗口宽 1280 高 800 比较合适
> 演示用户: 26 分 **650** (即 25 等位 ≈ 645, 26 位次约 4000)

---

## 准备 — 注入演示配置 (一次性)

打开 https://descreekert.github.io/zyhelper/ ,按 F12 → Console,粘贴运行下面脚本一键灌入演示配置:

```js
// === 演示用 配置 注入 ===
// 1. 用户分数
const ui = JSON.parse(localStorage.getItem('zyhelper_ui_v1') || '{}');
ui.myScore = 650;
ui.ratioChong = 0.25; ui.ratioWen = 0.45; ui.ratioBao = 0.30;
ui.totalVolunteers = 112;
// 冲稳保 范围 +20~+6 / +5~-5 / -6~-20  (注意: app 默认范围是 +6~+15 / -5~+5 / -20~-6,
//  这里改 ratio 不调范围, 范围在算法 commputeChongWenBao 里; 仅展示比例)
localStorage.setItem('zyhelper_ui_v1', JSON.stringify(ui));

// 2. 城市优先 (北上广深 > 南京 > 杭州 > 武汉 > 成都 > 其他)
const prio = JSON.parse(localStorage.getItem('zyhelper_priority_overrides_v1') || '{}');
prio.cities = [
  '北京','上海','广州','深圳',
  '南京',
  '杭州',
  '武汉',
  '成都',
];
// 3. 学校优先 (C9 → 985 → 211 → 双一流, 工科为主)
prio.schools = [
  // C9
  '清华大学','北京大学','上海交通大学','复旦大学','浙江大学','南京大学','中国科学技术大学','哈尔滨工业大学','西安交通大学',
  // 985 工科强校
  '北京航空航天大学','北京理工大学','同济大学','华中科技大学','东南大学','天津大学','大连理工大学','华南理工大学','北京邮电大学(可加)',
];
// 4. 专业类优先 (电子信息/计算机/自动化 三大类)
prio.majorClasses = ['电子信息类','计算机类','自动化类'];
// 5. 不接受 生化环材 — 用关键词过滤实现 (作为示例, 实际在筛选面板设置)
localStorage.setItem('zyhelper_priority_overrides_v1', JSON.stringify(prio));

// 6. 转专业目标 (默认已含, 这里追加 大数据/统计 相关)
const targets = JSON.parse(localStorage.getItem('zyhelper_transfer_targets_v1') || '[]');
['数据科学与大数据技术','大数据管理与应用','统计学','应用统计学']
  .forEach(m => { if (!targets.includes(m)) targets.push(m); });
localStorage.setItem('zyhelper_transfer_targets_v1', JSON.stringify(targets));

location.reload();
```

## 🎬 场景 1: 选志愿 — 筛选 / 排序 / 推荐

**用户故事**: "我考了 650 分,想看看 北上广深 + 南京 / 工科强校 / 计算机电子信息 自动化 类的专业有哪些。"

### 录制步骤 (约 30s 一帧 / 共 8-10 帧):

1. **首页全貌** — 顶部 `26 分数 650 / 26 位次 [自动]`,中间 `冲稳保占比 chips`,下方筛选面板 + 结果列表
2. **筛选: 学校优先 (前 18 档)** — 左侧筛选面板 → `🏫 学校优先` → 选 "第一档 1-18"。结果数从 ~10000 缩到 ~300
3. **筛选: 城市优先 (前 4 档)** — 左侧 `🏙 城市优先` → "第一档"。结果缩到 ~150
4. **筛选: 专业类优先 (前 3 类)** — 左侧 `📚 专业类优先` → 选 电子信息类/计算机类/自动化类。结果 ~80
5. **筛选: 关键字 — 排除"生化环材"** — 左侧关键字框 `-生物 -化学 -材料 -环境`。结果 ~50
6. **冲稳保 chip 点击 "稳"** — 顶部 `稳 N: 范围` chip 点击 → 结果只剩 稳档约 20 条
7. **切换三栏分桶视图** — 顶部 `⫷` 按钮 → 冲/稳/保 三栏并排,每栏带卡片
8. **点击一条 plan 看详情** — `📋 详情` 弹窗显示全部字段 (26/25 对比 + 历年 + 预测)
9. **点击 "➕ 志愿"** — 加入志愿单,右上角 `📋 志愿(N)` +1
10. **重复加几条** — 制造一张 20-30 项志愿单

输出: `docs/gifs/s1_select.gif`

## 🎬 场景 2: 报考志愿 — 多列表 / 排序锁定 / 导入导出

**用户故事**: "我把 80 项备选都选好了,需要按重要性排到正式 112 项志愿单里,还要锁住几个最想去的,最后导出官网格式上传。"

### 录制步骤 (~12 帧):

1. **切到志愿单视图** — 右上 `📋 志愿(N)` 点击,进入志愿表
2. **顶部下拉 — 新建志愿单 "650 分志愿"** — 顶部志愿单切换器,`新建` 按钮 → 输入"650 分志愿"
3. **从查询页加 30 个 plan** — 切回查询页,批量 ➕ 加入 (上一场景的成果)
4. **回志愿表** — 看到 30 项按 26 等位次自动排序
5. **整体上移一条** — 选中一行,点 `⤒` 置顶
6. **锁定某行** — 点行尾 `📌` → 该行变锁定状态 (有 📌 图标)
7. **批量锁全部** — 顶部 `锁所有` 按钮 → 所有 30 项全锁定
8. **导出 — 📤 导出 ▾ → CSV** — 下载 `志愿单_650 分志愿_2026-06-30.csv`
9. **打开 CSV 看一眼** — 系统打开,6 列 (序号/院校代号/院校名称/专业代号/专业名称/专业备注)
10. **导出 — 📤 导出 ▾ → HTML — 官网格式** — 下载辽宁招考网模板格式 HTML
11. **导入 — 📥 导入 ▾ → HTML — 官网/预选 (自动识别)** — 选刚下载的 HTML → 弹窗确认 → 选 "3. 新建"
12. **新志愿单出现,源顺序锁定 📌** — 顶部下拉切到新建的单,看到全部 30 项按源顺序排列,全部 📌 锁定

输出: `docs/gifs/s2_voluntary.gif`

## 🎬 场景 3: 分析 + PDF 报告

**用户故事**: "志愿单已经做好了 30 项,我想看看冲稳保比例是否合理,转专业风险有多大,各档我能上的最佳学校是哪些,最后导出 PDF 给家人看。"

### 录制步骤 (~15 帧):

1. **从志愿表点 `📊 分析`** — 进入分析页
2. **总览 — 冲/稳/保/外 四张卡** — 显示 N 项 / 26 招生 / 同分; 默认 `稳` 卡 + `保` 卡展开
3. **勾选 🔥 忽略专业热度 checkbox** — 看右侧概率从 65% → 67% 之类小变化
4. **🏆 各档录取率 Top 3 + 🎯 阈值 Top 5 表** — 滚动到 Top 汇总 区,看 3 张卡 + 11 行阈值表
5. **🎓 转专业风险扫描** — 滚动到转专业,看 3 张汇总卡 (✓ N / ⚠ N / ✗ N)
6. **点 ✎ 编辑清单** — 双栏 textarea 弹出,演示加一个目标 "电子信息工程" (其实已有)
7. **📈 录取趋势图** — 鼠标在曲线上移动,看 tooltip 显示具体志愿 + Δ 分位次
8. **拖动阈值滑杆 50% → 75%** — 看下方动态表条目数变化, 阈值线 横向上移
9. **点击概率分布柱 80-89 桶** — 阈值跳到 80
10. **冲稳保详细 chong 档展开** — 看详细表 (含 [新]/[变] badge + (估) + 🔥)
11. **滚动看 byScore / bySchool / byMajor / byCity 各种聚合表** — 每个表点行展开看明细
12. **回顶部, 点 `📄 PDF 报告`** — 切到 PDF 视图
13. **PDF 内容滚动 (section 1 → 15)** — 录 5-8 个关键 section: 总结建议 / 录取趋势 / 完整志愿表 / 学校总览 / 城市总览 / 转专业风险扫描 (含目标/可接受 清单)
14. **点 🖨 打印/保存 PDF** — 浏览器打印对话框,选另存为 PDF → 完成

输出: `docs/gifs/s3_analysis.gif`

## 录制小贴士

1. **浏览器** 窗口大小固定 (建议 1280×800),录前关掉书签栏 + 缩放 100%。
2. **LICEcap** 选 fps 8-10 即可 (gif 体积小); Kap 选 30fps + Lossy quality 50%。
3. **每步停留 1.5-2 秒** 让观众看清。
4. **鼠标光标** 用 LICEcap 自带的 显示选项 — 否则录出来看不到点哪。
5. **gif 体积优化**:
   ```bash
   gifsicle -O3 --lossy=80 -o out.gif in.gif
   # 或
   ffmpeg -i in.mov -vf "fps=10,scale=1024:-1:flags=lanczos,split[a][b];[a]palettegen[p];[b][p]paletteuse" out.gif
   ```
6. **加字幕** 推荐用 Captioner.app 或 ScreenFlow 后期叠字。
