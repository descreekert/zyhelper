## 3.16.2 收藏配额提示

### 设计

收藏栏增加汇总:

```
♥ 我的收藏 (84)
冲 28/28 ✓   稳 47/50 ⚠   保 30/34 ✓   未分类 0
[导出 CSV]  [按档导出]  [清空]
```

- 计算: 当前收藏中, 落入冲/稳/保 各档的数量
- 跟用户目标 `Math.round(totalVolunteers * ratio)` 对比:
  - 等于目标: ✓ 绿色
  - 少于目标: ⚠ 黄色 (`{cur}/{target}`)
  - 多于目标: ! 红色
- "未分类" = 不落入任何档的收藏 (停招 / 分数不在范围)

## 改动追溯

### 实施细节 (2026-06-23 完成)

#### Bug 修复 (优先做): priority filter 严格化

**根因**: applyFilters 用 `p.schoolPriority/cityPriority/majorPriority` 数值过滤。当 plan 的这些字段为 `null` 时, 条件 `field > max` 为 false → 不过滤, 该行通过。结果: 医学类等不在 priority.json top 16 的专业出现在 "Top 16 默认" 视图里。

**修复**:
- 新增 `priorityNamesInRange(items, nameKey, lo, hi)` — 从 priority.json 算出"允许的名字集合"
- 主 setup 加 `allowedSets` computed: `{schools, cities, majorClasses}` 各自的 Set
- `applyFilters(plans, filters, allowed)` 新增 `allowed` 参数
- 严格匹配: `if (allowed.majorClasses && !allowed.majorClasses.has(plan.majorClass)) continue;`
- `keywordCandidatePool` 也传 allowed
- 删除原 `p.majorPriority > max` 等基于 plan 字段的检查

**验证**: 输入 26 分 620 → 筛后 109 → **104** 条 (减 5 条非 top 16 专业类)

#### 3.16.1 三栏并排浏览

- 在视图切换按钮加 "⫷ 三栏" (没 cwb 时 disabled)
- `ResultList` 加 `viewMode='pane'` 模式
- `paneLists` computed: 当前 paged plans 按 planTier 分桶
- `paneCounts`: 各档数量
- 3 列 grid (md+ 才横向), 每列独立滚动 `.pane-col`
- 各列 header 显示 `冲/稳/保 + 数量 + 目标志愿数`
- 卡片复用 `PlanCard` (传 tier prop 让左边框着色)

#### 3.16.2 收藏配额提示

- FavoritesBar 加 `tierCounts()` 计算各档收藏数
- 上方 chip 行显示: `冲 N/T ✓⚠✗ | 稳 N/T | 保 N/T | 未分类 N`
- chip 配色:
  - 达标 (`N >= T`) → 加深, ✓
  - 部分 (`0 < N < T`) → 半透明, ⚠
  - 空 (`N === 0`) → 灰阶, ✗
- CSS: `.quota-chip.quota-{chong/wen/bao/other}` + 状态修饰

#### 3.16.3 按档分组志愿单导出

- FavoritesBar 加 `exportFavByTier()` 函数
- 收藏栏按钮: `导出 CSV` (原) / `按档导出` (新) / `清空`
- 导出格式:
  ```
  档,序号,学校,26招生专业,25招生专业,...,变化
  冲档 (28 条):
  冲档,1,武汉大学,...
  稳档 (47 条):
  稳档,1,...
  ```
- 每档内部重新计数 (1, 2, 3...), 便于对照志愿单填报

#### 其它顺手做

- DetailDrawer "26 变化" 行: 从纯字符串改为结构化渲染 (跟展开行格式一致)
- FavoritesBar 收藏卡片左边框按 tier 着色
- 修一处变量名 typo (planTierJS → planTier)
