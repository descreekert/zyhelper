# 冲稳保算法（志愿分布建议）

## 1. 目标

输入用户的 2026 高考分数（或位次），输出三档分数/位次区间 + 每档建议志愿数：

| 档 | 比例 | 默认志愿数 (112 总) |
|---|---:|---:|
| 冲 | 25% | 28 |
| 稳 | 45% | 50 |
| 保 | 30% | 34 |

总志愿数和比例都可配置。

## 2. 分数区间算法

按用户的策略（来自 xlsx 的 `志愿策略(分数)` 表，规则一致）：

```
输入: 用户分数 X (2026 实考分)

冲: 分数 [X + 6, X + 23]    宽度 18 分
稳: 分数 [X - 10, X + 5]    宽度 16 分
保: 分数 [X - 27, X - 11]   宽度 17 分

总跨度: X - 27 到 X + 23, 共 50 分
```

**冲稳/稳保 边界**：紧邻无 gap，无重叠（+5 / +6, -10 / -11）。

**验证用户数据**：600 分 →
- 冲 [606, 623] ✓
- 稳 [590, 605] ✓
- 保 [573, 589] ✓

## 3. 位次区间算法

xlsx 里 `等位分计算` 表已经给出每个 2026 分数对应的 25/24/23/22 等位次 + 平均等位次。

```
输入: 用户分数 X (2026)

第一步: 用"等位分计算"表查询各档分数边界对应的 25 等位次:
  r_chong_low  = 等位次(X + 6)
  r_chong_high = 等位次(X + 23)
  r_wen_low    = 等位次(X - 10)
  r_wen_high   = 等位次(X + 5)
  r_bao_low    = 等位次(X - 27)
  r_bao_high   = 等位次(X - 11)
  注: 分数高 → 位次小, 所以 r_*_high < r_*_low

第二步: 三档位次区间 (位次升序, 即 [更好的位次, 更差的位次]):
  冲位次: [r_chong_high, r_chong_low]   // 例: [1000, 1500]
  稳位次: [r_wen_high,   r_wen_low]      // 例: [1501, 3000]
  保位次: [r_bao_high,   r_bao_low]      // 例: [3001, 6000]
```

### 3.1 位次边界处理

位次是离散的（每个分数段的最低位次=该段累计人数）。**冲/稳交界**：

```
冲低界分数 = X + 6   → 位次 r_chong_low (=该段最低位次)
稳高界分数 = X + 5   → 位次 r_wen_high (=该段最低位次)

由于 X+6 和 X+5 是相邻分数, r_wen_high - r_chong_low = (X+5 段人数)
所以两段位次自然紧邻, 中间没有 gap。
```

不需要"-1"调整。直接用各分数的 25 等位次即可。

### 3.2 等位次的取值策略

xlsx 的 `等位分计算` 表给了 4 个等位次列：
- 25 等位次（去年）—— **默认推荐**
- 24 等位次
- 23 等位次
- 22 等位次（太久远，弃用）
- 平均等位次（25/24/23/22 简单平均）

**默认用 25 等位次**，理由：
- 距 2026 最近，物理类政策环境最相似
- 数据最稳定（24/23 期间有改革浮动）

**备选切换**（设置面板提供）：
- 切到 **平均等位次（25/24/23）**：抗噪性更好（避免某一年波动）
- 不允许 22（太老）

> 如果你以后觉得 平均等位次 更好，可一键切换。

### 3.3 位次区间的一种"更安全"替代

实践中考生分数有波动，**可在每档边界外做 ±缓冲**：

```
冲位次范围 = [r_chong_high - buffer, r_chong_low + buffer]
稳位次范围 = ...
保位次范围 = ...

buffer 推荐 = 各档跨度的 10%
```

但这会让相邻档重叠。本算法**默认不加 buffer**，保持档与档紧邻无重叠。需要时设置里加。

## 4. 志愿分配算法

```
输入:
  total_voluntary: 总志愿数 (默认 112)
  ratio: {冲: 0.25, 稳: 0.45, 保: 0.30}   // 默认

输出:
  num_chong = round(total_voluntary * ratio.冲)   = 28
  num_wen   = round(total_voluntary * ratio.稳)   = 50
  num_bao   = round(total_voluntary * ratio.保)   = 34
  
  // 校正: 总和等于 total_voluntary, 差额加到稳
  if (num_chong + num_wen + num_bao != total_voluntary) {
    num_wen += total_voluntary - num_chong - num_wen - num_bao;
  }
```

四舍五入后总和可能差 ±1，差额加到"稳"档（容量最大，最不敏感）。

### 5.2 三档分别可"应用到主筛选"

点"冲档应用"：自动把分数滑块设到 `[X+6, X+23]`，位次滑块设到对应区间。结果区刷新出冲档候选学校。

### 5.3 "一键三档浏览"模式

按下"冲稳保浏览"按钮：
- 主结果区切到三栏布局（冲 / 稳 / 保）
- 每栏内是该档的候选招生计划（独立排序、独立分页）
- 用户可在每栏单独勾收藏

### 5.4 志愿单导出

收藏 ≥ 1 的话，"导出志愿单"按钮：
- 按冲稳保分组（自动识别每条收藏属于哪一档）
- 导出 Excel：含学校 / 专业 / 25 参考分 / 26 计划数 / 学费 / 评估 / 选科要求
- 文件名：`志愿单_{用户分数}_{日期}.xlsx`

### 5.5 收藏数提醒

收藏栏标题：`我的收藏 (28/28 冲 ✓ | 47/50 稳 ⚠ | 30/34 保 ⚠)`

数字达标 / 接近目标显示状态色，超额报警。

## 6. 边界情况

| 情况 | 处理 |
|---|---|
| 用户分数低于 等位分计算表 最低分 | 等位次取表中最低分对应位次，外推（warning 提示）|
| 用户分数超出表最高分 | 同理外推 |
| 用户只输入位次不输分数 | 反向查询：先用 25 一分一段算出对应 2026 分数（用等位次反查），再走标准流程 |
| 比例和 ≠ 1 | 警告，强制重设为 0.25/0.45/0.30 |
| 总志愿数 = 0 | 警告，默认 112 |

## 7. 不做的（明确边界）

- ❌ 智能"是否能录取"概率预测（统计学复杂，且依赖更多数据）
- ❌ 跨年趋势的复杂回归（够了，等位次已经做了大部分活）
- ❌ 同分位"专业级冲稳保"（每个学校的专业冷热不同，复杂）
