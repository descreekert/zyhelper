# 功能需求文档

## 1. 目标用户

考生本人 + 家长。物理类，化学/政治选考，无生物/地理。**单一用户场景**（不需要多用户/权限管理）。

## 2. 用户痛点（要解决的核心问题）

| # | 痛点 | 解决方案 |
|---|---|---|
| 1 | Excel 81 列太宽，关键信息找不到 | 重要字段做成卡片，次要字段在详情弹窗 |
| 2 | 反复改 B6/G6 改不动 | 滑块/可调输入，结果即时刷新 |
| 3 | 多校多专业对比难 | "加入对比"按钮，最多 4 项并排卡片 |
| 4 | 学科评估筛选不直观 | 多选 chip（A+/A/A-/B+/B/...）|
| 5 | 分数位次想看实时定位 | 输入 X 分，弹"对应位次 N 名" + 一分一段表 |

## 3. 功能清单（按优先级）

### P0 - 必做

#### 3.1 数据加载
- 启动时加载 `data/plans.json`（约 12.7k 行招生计划）
- 加载 `data/score_rank.json`（一分一段表）
- Loading 状态提示，加载完成后展示首屏

#### 3.2 主筛选侧栏（左侧或顶部抽屉）

**基础筛选** (V2 — User Review 1/2/3/6 改造后)

| 控件 | 字段 | 行为 |
|---|---|---|
| **3 段范围 (冲红/稳黄/保绿)** | 25参考最低分 | 由顶部"我的分数"自动填充, 用户可手改; 复选框启用 |
| **3 段范围 (冲红/稳黄/保绿)** | 25参考最低位次 | 同上 (默认关闭, 通过复选框启用) |
| 单向滑块 | 学费上限 | 默认 `100000` |
| **双端范围 + 三档预设** | 🏫 学校优先 (排序) | 默认 [1, 50], 预设: 第一档 1-18 / 第二档 20-50 / 第三档 50+ / 全部 |
| **滑杆 + chip 多选** | 🏙 城市优先 | 默认 ≤18 (主流), 预设 Top 10/18/30 |
| **滑杆 + chip 多选** | 📚 专业类优先 | 默认 ≤16 (断层位置), 预设 Top 8/16/25 |
| 多选 chips | 学科评估接受范围 | 默认全选 |
| 多选 chips | 院校标签 | 985/211/双一流/国重点/省重点/保研资格 |
| 开关 | 含停招 | 默认 关 |
| 开关 | 含中外合作办学 | 默认 开 |
| 开关 | 仅"有 25 参考" | 默认 关 |

**V1→V2 移除**: ~~门类 chip~~ / ~~所在省 chip~~ / ~~城市水平 chip~~ / ~~学校优先单端滑杆~~ / ~~专业类优先单端滑杆~~ / ~~25参考分单一范围~~ — 被优先次序组件 + 3 段范围替代

**优先次序数据源**: `data/priority.json` (来自 xlsx 排名 sheet, 城市 79 + 学校 328 + 专业类 95)

**搜索框** (V3 — User Review 5 + Round 3 Item 7)

- 智能补全下拉, 三个维度分组展示: 🏫 学校 / 📚 专业类 / 📖 专业
- 选中候选 → 转为 chip (强等于过滤); 回车 → 模糊关键词
- **候选范围 = 当前其他筛选条件结果集** (避免"补全看到, 选了无结果"困境)
- 下拉顶部显示 `📍 在当前筛选 N 条结果内补全`, N 动态变化
- 5 个微妙点处理: 见 `06_user_review_2026-06-23.md` + `07_user_review_2026-06-23_round3.md`

#### 3.3 结果区

**布局选项** (V4 — User Review Round 4 后)

最新版表格特性:
- 表头 sticky (滚动表格内容时, 表头固定)
- 表头 3 子元素: `⠿` 拖动手柄 + 文字点击排序 + 右边缘拖宽 (互不干扰)
- 默认列顺序: 档/城市/院校/专业/计划/学制/学费/变化/25分数/25位次/可信度/学科评估/软科评级/专业备注/专业保研/操作
- 行展开时主行字段不再截断 (持久 hover 效果)
- 展开行 5 块布局 (section 卡片): 学校 / 专业 / 25vs26 对比 / 预测 / 变化
- 25 vs 26 对比表多年并排 (含 22-24 历年)

筛选侧栏:
- 学校优先 默认第一档 [1, 18], 城市优先 默认 ≤ 10
- 预设 chip 点击后蓝底白字高亮

操作:
- 顶栏左 `≡` 收起侧栏 / 收起态左侧浮 `›` 按钮展开
- 顶栏右 `🔄 刷新` 重跑 load
- 顶栏右 `⚙ 列` 列设置 panel, 点空白处自动关闭

#### 3.4 详情弹窗

点"详情"或卡片本身，弹出全字段（81 列分组）：

```
基本信息: 院校层级、所在省、城市水平、办学性质、主管部门、保研率/升学率
26 招生: 专业名 / 所含专业 / 选科 / 学制 / 学费 / 人数 / 变化
评估排名: 学科评估 / 软科专业排名 / 软科硕士排名
保研详情: 专业保研率（含学院）
专业备注: 校区/分流/转专业 政策
25 参考: 来源 + 可信度 (说明哪些 25 数据被用作参考)
25 实际数据: 25 招生专业 / 录取 (人数/最低分/最低位次/线差)
历年录取: 24/23/22 三年数据

```

#### 3.5 对比模式

- 卡片上"加入对比"按钮，最多 4 个
- 对比页：4 列卡片并排，关键字段对齐（学费、评估、保研率、25 分、预测分）
- 数字差异自动高亮（绿/红表示哪个高/低）

#### 3.6 排序

- 排序选项：25参考分降序 / 25参考位次升序 / 校优先升序 / 学费升序 / 城市优先 / 院校层级
- 默认：25参考分降序

## 4. 用户故事示例

### Story A："我想看 640-650 分能上什么"
1. 打开 web app，左侧栏分数范围拖到 `640-650`
2. 立即显示符合的所有招生计划（按分数降序）
3. 翻看几个学校，发现"哈尔滨工业大学 工科试验班"
4. 点"详情"，看专业备注 / 选科要求等细节
5. 点"加入对比"加进对比栏
6. 同样看几个其他学校，加入对比
7. 切到对比视图，4 个并排对比，决定填报顺序

### Story B："我位次 8000，想锁定 1500-2500 段做冲一冲"
1. 顶部位次定位器输入 `8000`，显示对应分数 ~625
2. 改用位次区间：`[1500, 2500]`（冲一冲，比我高 5500-6500 名）
3. 学校优先放宽到 ≤50（看更多学校）
4. 院校层级勾选 985 / 211
5. 看结果，选择前 5 名学校的目标专业，加入收藏

### Story C："我想专门看南京的院校"
1. 所在省勾选"江苏省"，城市勾选"南京"
2. 院校层级 985 / 211
3. 浏览全南京 985/211 的招生计划

## 5. 非功能需求

| 项 | 要求 |
|---|---|
| 响应时间 | 筛选响应 < 200ms（不能感受到卡顿）|
| 数据规模 | 12k 行招生计划，~5-10 MB JSON |
| 兼容性 | Chrome / Edge / Safari 最新版（家用就够）|
| 可访问性 | 桌面优先，平板/手机能用即可 |
| 持久化 | localStorage（收藏/预设/主题）|
| 离线 | 一次加载后离线可用（不依赖外网 CDN，必要时打包） |
| 部署 | `python3 -m http.server` 一行启动 |
