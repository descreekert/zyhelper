## 详细设计

### 1. 专业类优先（取代"门类"）

**需求**:
- 滑杆控制专业类的优先值范围（默认 ≤16）
- 下方 chip 列出该范围内所有专业类（默认显示排序 1-16）
- 滑杆扩大 → chip 自动补全；滑杆缩小 → chip 自动减少
- 用户可单独点 chip 取消某个专业类（缩小集合，但不改变滑杆）
- 显示格式: `[工学] 计算机类 (3)` — 带门类前缀和排序号

**最佳实践参考**:
- 类似 LinkedIn / Notion 的 "Range + Tag Override" 模式
- Material UI Range Filter / Apple HIG Token Selector
- 关键: **滑杆 = 候选集合**，**chip = 实际筛选集合**

**实现方案**:
- 滑杆值 `majorClassPriorityMax = 16`
- 候选集 = `majorClassPriority.filter(c => c.sort <= max)`
- 实际筛选集 `selectedClasses: Set<className>` = 默认 = 候选集
- 滑杆变化 → 重算候选集 → 同步更新筛选集（新增 chip 自动选中）
- chip 点击 → 仅修改筛选集
- 筛选逻辑: `plan.majorClass ∈ selectedClasses`

### 2. 城市优先（取代"城市水平"）

同 #1 模式，默认 `cityPriorityMax = 18`（主流城市断层位置）。
筛选逻辑: `plan.city ∈ selectedCities`

### 3. 学校优先（取代"所在省"）

**双端滑杆**: `[lo, hi]`，默认 `[1, 50]`（覆盖 user 的"第一档 + 第二档"）。

**chip 显示**:
- 候选 = `schoolPriority.filter(s => lo ≤ s.sort ≤ hi)`
- 显示格式: `[985] 南京大学 (1)`
- 排序 0 用"档段分隔线": `── 第二档 (20-50) ──`

**预设按钮**（chip 上方）:
- `第一档 (1-18)` / `第二档 (20-50)` / `第三档 (50+)` / `全部`

筛选逻辑: `plan.schoolName ∈ selectedSchools`

### 4. 26 分数 → 25 等位分换算（核心改动）

**当前 bug**: `computeChongWenBao(score26)` 直接对 26 分数加减 → 但 25 数据是用 25 标尺，**不能直接比较**！

**正确逻辑**:
```
1. 用户输入 26 分数 X
2. 查 equivalent 表: X (26) → Y (25 等位分), R (25 等位次)
3. 冲稳保区间基于 Y 计算:
   - 冲: Y + [6, 23]
   - 稳: Y + [-10, +5]
   - 保: Y + [-27, -11]
4. 用 25 一分一段反查每个区间端点的 25 位次范围
5. 显示: "26 分 X = 25 等位 Y / 等位次 R"
       "冲档: 25 分 [Y+6, Y+23] / 位次 [R+a, R+b]"
```

**等位次基准**支持切换:
- 默认 `25等位次` (rank25 字段)
- 备选 `平均等位次` (rankAvg)
- 备选 `24等位次` (rank24)
- 备选 `23等位次` (rank23)

### 5. 关键词智能补全

**触发**: 用户在搜索框输入 ≥1 字符 → 下拉列表显示候选

**候选来源 + 优先级**:
1. 学校名（候选 = 所有 schoolName 字符串包含输入）
2. 专业类（候选 = 排名 sheet 中所有专业类）
3. 专业名（候选 = 所有 majorName26 / majorName25）

**展示**:
```
┌─────────────────────────┐
│ 🏫 学校 (3)             │
│   南京大学               │
│   南京邮电大学            │
│   南京航空航天大学        │
│ 📚 专业类 (1)            │
│   电子信息类             │
│ 📖 专业 (5)              │
│   电子信息工程            │
│   ...                   │
└─────────────────────────┘
```

**选中行为**:
- 选学校 → `filters.schoolNameKw = X`，只筛该学校
- 选专业类 → `filters.majorClassKw = X`，只筛该专业类
- 选专业 → `filters.majorNameKw = X`，模糊匹配
- 直接回车 → 三个维度都模糊匹配（当前的 `keyword`）

**实现细节**:
- 用原生 `<datalist>` 还是自定义下拉？→ **自定义下拉**（datalist 不支持分组）
- 候选总数限制每组 8 条
- 字符匹配 = 子串匹配（不做拼音）

### 6. 25 参考范围 3 段冲稳保化

**侧栏新版**:
```
[ ] 启用分数范围筛选
   冲: [625, 642]  ← 自动填 (来自顶部 26 分)
   稳: [609, 624]
   保: [592, 608]

[ ] 启用位次范围筛选
   冲: [3200, 5500]
   稳: [5501, 8500]
   保: [8501, 13000]
```

**筛选逻辑（关键）**:
- 取 3 段的**并集** = 一个 plan 落在冲 OR 稳 OR 保 中任一即通过
- 两组复选框（分数 / 位次）都启用 → plan 必须同时满足分数并集 AND 位次并集
- 两组都不启用 → 不按这两个维度筛选

**颜色标记**:
- 表格行 / 卡片左边框:
  - 落在 "冲" → 红色 (`#fee2e2` 背景 / `#dc2626` 边)
  - 落在 "稳" → 黄色 (`#fef3c7` / `#d97706`)
  - 落在 "保" → 绿色 (`#dcfce7` / `#16a34a`)
- 优先用分数判档；停招行用 25 实际分数判档（非 25 参考分）
- 不在任何档 → 无颜色（中性）

### 7. 表格列宽 + 分页

- 列宽: CSS `resize: horizontal; overflow: hidden` 给 th + `min-width`
  - 或用 [Sortable.js](https://sortablejs.github.io/) 系的列宽插件
- 默认不分页（`pageSize: Infinity` 或 `null`）
- 每页条数选择器: `[全部, 50, 100, 200, 500]`，放在排序下拉旁

### 8. 表格列追加 + 展开

新增 3 列: `26变化` `专业备注` `软科评级`
字体: 12px → 11px

**展开行**:
- 表格行点击 → 当前行下方插入 detail row (colspan 跨满)
- 展开行内容: 简化版"详情抽屉"（学校 + 评估 + 25 参考 + 历年）
- 再点击 → 折叠
- 也可以保留"双击→侧边抽屉"作为备选

### 9. 表格列删减

- 删除"选科"列（基本都是"化学/政治/不限"）
- "学制"字段格式: `四年` → `4`, `五年` → `5`, `八年医` → `8`

### 10. 默认视图 + 卡片增强

- 默认 `viewMode = 'table'`
- 卡片视图追加"变化"字段（与表格同列同步显示）

### Item 5 — KeywordAutocomplete 组件

4 个微妙点的处理:

1. **"已选 chip" vs "未选输入"** 区分: filters.pickedSchool/pickedMajorClass/pickedMajorName (强等于) + filters.keyword (模糊兜底)。回车若有高亮候选则选中, 否则把输入作为模糊关键词。
2. **专业类与专业名重叠**: 分组下拉, 用 emoji + label 区分 🏫 学校 / 📚 专业类 / 📖 专业。
3. **多维度 AND**: pickedSchool / pickedMajorClass / pickedMajorName 各自独立, 三者均参与 AND（同一维度本版单选）。
4. **mousedown 拦截**: 候选项用 `@mousedown.prevent="pickItem(item)"` 替代 click, 避免 input blur 先关掉下拉。setTimeout 150ms 兜底关闭。

### Item 6 — 3 段冲稳保范围筛选 + 颜色标记

**状态形状**:
- `enableScoreRange` / `enableRankRange` 复选框
- `scoreRanges: [{tier, low, high}]` × 3 段 (cwb 变化自动填充, 用户可手改)
- `rankRanges: [{tier, low, high}]` × 3 段

**筛选逻辑**: 3 段取并集 (任一档段匹配即通过), 两个范围都启用时取交集 (AND)

**颜色 (CSS)**:
- `tier-row-chong/wen/bao` 行背景 + hover 加深
